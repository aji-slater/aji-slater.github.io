Ruby's Own Little Card Catalogs

Or: The Digital Librarian Strikes Again!

September 11, 2015

Image by Megan Amaral

Programs need ways to store information. Variables are objects which can store one piece of information, whether that is an integer, a floating point number, a boolean or a string.

Even more versatile than variables are arrays and hashes. Where variables are a single stored object, arrays and hashes are indexed lists of objects.

An array is identified by square brackets, with commas separating each object stored therein, and are held in variables. Each cell of the array is assigned an integer to identify it for quick and simple access. Be careful though, cells are labeled starting with 0 and not 1.

my_array = [ "A string", 45, true, 3.14159 ]

A hash is very similar to an array, but instead of being indexed simply with integers, the hash can be indexed with any kind of object you want. Although there are several methods, let's just take a look at one way we would create a simple hash.

eric_sogard = {"team": "The Oakland Athletics"}

Like other variables, a hash is first created by giving it a name to the left of an equals sign. In the example above, we've only defined one pair of values. The first entry is called the key, and the second the value. The ability to index information with more than just whole numbers opens up a whole host of possibilities for use.

eric_sogard[:team]

This is the syntax that is required to call a value from a hash based on its key. Much like you would call the value from an array with its index number, you can use a much more descriptive key with a hash. Think of the pairs like the stats on the back of a baseball card.

eric_sogard = Hash.new
eric_sogard["Team"] = "The Oakland Athletics"
eric_sogard["Position"] = "2B/SS"
eric_sogard["Nickname"] = "Nerd Power"

Notice the new way that I entered the information into the hash. I began by using Hash.new to let ruby know to make an empty hash with the name eric_sogard. Each subsequent line is using very similar syntax as what we used to call the method in order to insert a value there.

It's important to remember that while hashes are more flexible and key/value pairing allows for more information that is more semantic to access; arrays are much more efficient to run.

I hope you were able to learn something new from this brief introduction to container objects in Ruby. See you next time!
